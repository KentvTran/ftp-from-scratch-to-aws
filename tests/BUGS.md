# Bug Tracking Document

This document tracks bugs discovered during testing with reproduction steps.

## Bug Report Template

For each bug, document:
- **Bug ID**: Unique identifier (e.g., BUG-001)
- **Severity**: Critical / High / Medium / Low
- **Status**: Open / In Progress / Fixed / Won't Fix
- **Description**: Clear description of the issue
- **Reproduction Steps**: Step-by-step instructions to reproduce
- **Expected Behavior**: What should happen
- **Actual Behavior**: What actually happens
- **Environment**: Server version, OS, Python version, etc.
- **Additional Notes**: Screenshots, logs, workarounds, etc.
